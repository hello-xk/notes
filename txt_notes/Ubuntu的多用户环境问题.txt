下面是一个奇怪的现象，
    创建一个用户，然后登录，发现无法进入桌面。为什么新建的用户也无法登录呢？？？
    
    回到自己创建的步骤：
    sudo useradd kuang      //创建一个用户，默认创建该分组
    sudo passwd kuang       //为用户分配密码
    mkdir /home/kuang       //为用户创建目录，但这里没有分配权限，使其具备访问权，故无法登录的！
    
    然后登录。

如果对linux的多用户环境，权限问题比较敏感的话，一下子就会注意到，/home/kuang这个目录并不是由kuang自身创建的，所以kuang并不对这个目录有读写权限！！这就是用户无法登录的原因。因此，一个新用户无法登录的原因有： 1，用户目录kuang不存在。2，用户对kuang目录不具有读写权限。3，前面的条件都符合，用户初始化时自动执行的程序的初始化出错，导致不能进入桌面。（今天出现的问题是第三点）

修复： 要用户能登录，需要为用户创建一个能够访问的文件夹，和为此用户分配一个密码。
    sudo chown kuang:kuang kuang    // owner:usergroup file 
    sudo chmod 755 kuang            // 分配文件夹权限
现在可以登录了！

要创建更加丰富的帐号，自己查阅相关资料。sudo useradd -r kuang可以创建kuang为系统用户。

登录到另一个用户，在ema用户安装的软件其实一个都不能用的。
上面引申出来的问题是，多用户环境。

一个用户，会有一个用户目录，如ema用户的/home/ema/。这个目录和用户息息相关。目录下一些重要的文件夹：
.config     用户软件的配置。比如，chromium, ibus, gedit, nautilus, software-center等等。
        //如果把这个文件夹删除，chromium的登录信息会丢失，ibus的输入法可能不能使用，gedit的历史等等。
.cache      缓存目录。比如，chromium，unity，ibus，等等。
        //至于为什么.cache这个目录会影响到进入桌面，这就很难理解了。
.gconf      gnome2的配置文件

.XXXX   // XXXX软件的目录，包含软件的配置文件，自身需要存放文件的目录等等。
        比如： .ssh 是ssh的key目录； .MyEclipse 是MyEclipse的目录； .vim 是vim的目录。
